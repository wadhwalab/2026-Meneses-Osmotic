# Osmotic Stress Triggers Fast and Reversible PMF Collapse in *Escherichia coli*

**Meneses, Javi, Dudebout, Belser, Yang & Wadhwa (2026)**

This repository contains the data, analysis code, and manuscript source for the paper studying how hyperosmotic shock rapidly and reversibly collapses the proton-motive force (PMF) in *E. coli*, as reported by the bacterial flagellar motor and the Nernstian fluorescent dye TMRM.

## Analysis Workflow

The analysis is organized into three parallel pipelines — **Bead/Tethered-Cell Assay**, **TMRM Fluorescence**, and **Cell Area** — that each follow the same conceptual pattern: *image segmentation → per-cell time series extraction → ΔF/F₀ (or ΔA/A₀) normalization → population statistics and plotting*.

### Overview

```
Raw video (.avi / .nd2)
        │
        ▼
  [Step 1] Cell/bead detection & segmentation
        │
        ▼
  [Step 2] Extract per-cell time series (rotation frequency, fluorescence, or area)
        │
        ▼
  [Step 3] Merge multiple recordings into one CSV (if needed)
        │
        ▼
  [Step 4] Compute ΔF/F₀ (or ΔA/A₀) relative to pre-shock baseline
        │
        ▼
  [Step 5] Plot individual traces + population mean ± SEM
```

### Pipeline 1 — Bead / Tethered-Cell Assay (Motor Speed)

These scripts measure the rotation frequency of a polystyrene bead (or whole tethered cell) attached to the flagellar motor as a proxy for PMF.

#### Step 1 — `code/bead-assay/Detection.py`

**Input:** `.avi` video of a rotating bead/tethered cell.

**What it does:**
1. Prompts the user to select a video file and a results directory via GUI.
2. Extracts every frame as a TIFF image.
3. Applies Gaussian blur → multi-class Otsu thresholding to identify the bead/cell in each frame.
4. Uses `skimage.measure.regionprops` to record the centroid (`x`, `y`), area, and axis lengths for each frame.
5. Fits a circle to overlapping 10-second windows of centroid positions (using the Karimäki–Mühlegger–Hautamäki algorithm from `circle_fit`) to estimate the center of rotation.
6. Saves `frame-properties-df.csv` (centroid data), `center-coordinates.csv`, and a trajectory scatter plot.

**Key parameters:** `FPS` (frames per second), `windowsize = FPS × 10`.

**Output:** Per-video folder containing `frame-properties-df.csv` and `center-coordinates.csv`.

---

#### Step 2 — `code/bead-assay/Rotfreq.py`

**Input:** `params.csv` generated by `Detection.py` (contains the results directory path and FPS).

**What it does:**
1. Reads the centroid and center-of-rotation data for every video sub-folder.
2. Corrects the center of rotation: if the bead's displacement over 10 frames is less than half the cell diameter, that center estimate is replaced with the previous frame's value (removes jitter when the motor is paused).
3. Computes instantaneous rotation frequency:
   - Calculates the angle `θ = arctan2(y − y_c, x − x_c)` and unwraps it over time.
   - `f = FPS × Δθ / (2π)`
4. Applies a median filter (kernel size 51 frames) to smooth the frequency trace.
5. Saves `freq.csv` (filtered frequency) and `freq-plot.pdf`.

> **Note:** For clockwise (CW) rotation, multiply line 72 by −1 before running.

**Output:** Smoothed rotation-frequency time series per cell.

---

#### Downstream notebooks — `code/bead-assay/`

| Notebook | Purpose |
|---|---|
| `plot-and-curve-fit.ipynb` | Loads all frequency CSVs, normalizes to pre-shock baseline, overlays individual traces, and plots population mean ± SEM for each osmolarity condition. |
| `adaption-curve-fitting.ipynb` | Fits a single-exponential recovery curve to the normalized motor speed during the post-shock adaptation period; extracts time constant τ and onset time t₀ per condition. |
| `sucrose-shock-plotting.ipynb` | Specialized plotting for sucrose-shock experiments; generates the combined speed-decrease figures used in the manuscript. |

`Viscosity.py` overlays a viscosity-correction dotted line on the normalized speed plots to confirm that the observed speed reduction exceeds what viscosity alone would predict.

### Pipeline 2 — TMRM Fluorescence (PMF Reporter)

TMRM (tetramethylrhodamine methyl ester) is a Nernstian dye whose intracellular fluorescence is proportional to the membrane potential. These scripts process two-channel ND2 microscopy files (phase contrast + TMRM fluorescence).

#### Step 1 — `code/TMRM-anaylsis/018_New_Luis4_TWO-CHANNEL_NewEdition.py`

**Input:** `.nd2` file with two channels — `c=0` phase contrast, `c=1` TMRM fluorescence.

**What it does:**
1. Opens the ND2 file with `nd2reader`.
2. Displays the first phase-contrast frame and prompts the user to draw rectangular ROIs around individual cells using `matplotlib.ginput`.
3. For every ROI in every time frame:
   - Segments the cell boundary from the phase image using Gaussian smoothing → Otsu threshold → size filter (keeps only the largest object above `min_area = 100 px`).
   - Records the cell **area** (pixel count of the mask).
   - Computes the **mean TMRM fluorescence** inside the cell mask.
   - Computes the **background fluorescence** outside a dilated version of the mask.
4. Outputs wide-format CSVs (one column per ROI) for areas and fluorescence, plus a normalized ratio CSV `csv555_LONG_RatioNorm_Fluor_mean_018_.csv` in long format (`track_id`, `frame`, `fluorescence`).
5. Saves segmentation-overlay images for visual quality control.

**Key parameters:** `SIGMA = 2.5` (Gaussian sigma), `min_area = 100`, `GrowMask = 4` (background dilation), `frame_interval_s`.

---

#### Step 2 — `code/TMRM-anaylsis/020_Concatenate_CSVs_in_Folder.py`

**Input:** Folder of long-format CSVs from Script 018 (one CSV per recording session).

**What it does:** Reads all CSVs, re-indexes `track_id` to be globally unique across files (adds an offset so IDs from different files do not collide), and concatenates them into a single `merged_fluorescence_020.csv`.

---

#### Step 3 — `code/TMRM-anaylsis/019 FJF_Fo_TMRM-post-analysis-DeltaF_Fo.py`

**Input:** A single merged long-format CSV.

**What it does:**
1. For each `track_id`, extracts a pre-shock baseline window (`shock_frame − baseline_window` to `shock_frame`).
2. Computes F₀ as the mean fluorescence in that window (requires ≥ 3 baseline points).
3. Computes: `ΔF/F₀ = (F − F₀) / (F₀ − 1)` for every frame.
4. Saves an updated CSV (`cell_detections_dff_019_.csv`) and two plots:
   - `single_cell_dff_traces.png` — individual cell traces.
   - `population_dff_mean_sem.png` — population mean ± SEM.

**Key parameters:** `shock_frame = 35`, `baseline_window = 10`, `frame_interval_s = 5`.

---

#### Step 4 — `code/TMRM-anaylsis/021_MergeMultiDFF.py`

**Input:** Folder of `_dff.csv` files produced by Script 019, one per osmolarity condition.

**What it does:** Loads all ΔF/F₀ CSVs, computes per-condition population statistics (mean ± SEM over all `time_s` values), and generates a single combined multi-condition plot with Control first, then ascending osmolarity concentrations (e.g., 200 mM, 300 mM, 400 mM, 500 mM).

### Pipeline 3 — Cell Area Change

These scripts track changes in cell cross-sectional area upon osmotic shock (cell shrinkage) using phase-contrast images only.

#### Step 1 — `code/Cell-area-anaylsis/018-4 New_Luis5_One_Channel_AREA - LONG_CSV.py`

**Input:** Single-channel `.nd2` phase-contrast video.

**What it does:**
1. Displays the first frame and prompts the user to draw rectangular ROIs around individual cells.
2. For each ROI in each frame, segments the cell using a **Difference of Gaussians (DoG)** filter (σ_A = 1.0, σ_B = 4.0) followed by Otsu thresholding, binary closing, and hole-filling.
3. Records the cell **area** in pixels.
4. Saves both a wide-format CSV (one column per ROI) and a long-format CSV (`track_id`, `frame`, `area`) for downstream analysis.

**Key parameters:** `SIGMA_A = 1.0`, `SIGMA_B = 4.0`, `MIN_AREA = 50`, `MAX_AREA_PIXELS = 200`, `frame_interval_s = 1`.

---

#### Step 2 — `code/Cell-area-anaylsis/021_Concatenate_Area_CSVs_in_Folder.py`

**Input:** Folder of long-format area CSVs from Script 018-4 (one CSV per recording session).

**What it does:** Same logic as the TMRM concatenation script — re-indexes `track_id` globally and concatenates all files into `merged_fluorescence_020.csv`.

---

#### Step 3 — `code/Cell-area-anaylsis/019-2 F_Fo_for_AREA_TMRM.py`

**Input:** A single merged long-format area CSV.

**What it does:**
1. For each `track_id`, computes A₀ = mean area over the pre-shock baseline window.
2. Computes: `ΔA/A₀ = (A − A₀) / A₀` for every frame.
3. Saves an updated CSV and individual/population plots.

---

#### Step 4 — `code/Cell-area-anaylsis/021-2 MergeMultiDFF_AREA__.py`

**Input:** Folder of `_dff.csv` files (one per osmolarity condition).

**What it does:** Same multi-condition aggregation and plotting as the TMRM equivalent, but uses ΔA/A₀ and labels the y-axis accordingly.

## Key Experimental Parameters

| Parameter | Value | Description |
|---|---|---|
| `shock_frame` | 35 | Frame index at which osmotic shock is applied |
| `baseline_window` | 10 frames | Pre-shock frames used to compute F₀ / A₀ |
| `frame_interval_s` | 5 s (TMRM) / 1 s (area) | Seconds per frame |
| Osmolyte conditions | Control, 100, 200, 300, 400, 500 mM | Sorbitol, sucrose, or sodium-based solutions |
| Motor assay FPS | User-defined | Frames per second of bead-assay video |

## Data

The `data/` directory contains processed or summary data included in the repository for reproducibility:

- `data/Bead-assays/` — Per-cell rotation frequency CSVs organized by condition (Sucrose, Sorbitol, Sodium, Clockwise, Adaptation, 100 mM, 20 mM + osmoprotectants).
- `data/TMRM-results/` — Merged ΔF/F₀ CSVs per osmolarity condition (200–500 mM + control).
- `data/Cell-area/` — Merged ΔA/A₀ CSVs per osmolarity condition (200–500 mM + control).
- `data/Osmolarity-readings/` — Raw osmometer measurements for each solution used (`sucrose.xlsx`, `sorbitol.xlsx`, `sodium.xlsx`).
- `data/Viscosity/` — Viscosity correction data for sorbitol and sucrose solutions.

Raw microscopy videos (`.nd2`, `.avi`) are not included due to file size.
